Mysql 主从同步

mysql主从同步调研：

1. pre check:

查询slave服务器的大小写敏感，1为大小写不敏感 0则反之

查询master 要同步的DB字符集, slave 必须相同 否则会报1667

查询主库的binlog得到mysql-bin.002884 和 position 362448
show master status;

2. 在slave服务器，修改：/etc/my.cnf.

a. 增加这几句：

server-id = 2
expire_logs_days = 10
max_binlog_size = 100M
replicate-do-db = ajmide_stat
replicate-ignore-db = mysql
replicate-ignore-db = information_schema
replicate-ignore-db = preformance_schema
slave-skip-errors=all

#GTID
gtid_mode=on
enforce_gtid_consistency=on
binlog_format=row
log-slave-updates=1

b. 修改后，重启mysql.然后登录mysql

issue tracker:

  1. 检查表CHARSET,或在创建database的时候指定默认的字符集（CREATE DATABASE ajmide_stat DEFAULT CHARACTER SET utf8; ）
    如需改变整个mysql服务器的字符集 则需将在/etc/my.cnf 新加 default-character-set=utf8
    mysql 5.7版本选择 character_set_server=utf8
  1. 检查大小写敏感问题 ， master建表时如有大小写，slave也会实时完全同步过来， 但是dml的sql语句会全部转为小写（引号内的不算）

Hive合并小文件的方法

title: Hive合并小文件的方法
date: 2018-11-26 20:38:17
updated: 2018-11-26 20:38:17
categories: Platform
tags: [HDP]

hive表中大量的小文件一直是影响生产效率的问题之一，合并的小文件的工作也是迫在眉睫，百度了很久，网上提供的解决方案千篇一律，无外乎hive内置的功能，当然也是有用的。其他的方法嘛，Google了一下，发现有现成的工具可以合并。但根本的解决办法是如何避免产生小文件，下面列举的方法只能作为Workarond。

方法列举如下：

1. 把有大量小文件的表插到临时表中，drop源表+rename临时表 或 清空+回插到源表 (需要定期维护) 或者使用外部表，手动合并小文件
2. 使用filecrush先合并到临时目录，再load+overwrite源表。（数据在合并时是可用的，load回去速度也很快）

还有许多其他方法参考 搜索引擎 ，如hive 调参， 使用hadoop-streaming.jar去合并

Filecrush Demo

如需更新更多功能请下载最新版，请参考如下步骤（亲测打成jar包后运行失败- -!）

  1. 去下载源码 filecrush-2.2.2
  2. 按教程打成jar包filecrush-2.2.2-SNAPSHOT.jar filecrush2.2源码编译教程
  3. 用法参见第一步
  4. 老版本下载地址http://www.jointhegrid.com/hadoop_filecrush/ 具体用法自行搜索或反编译
